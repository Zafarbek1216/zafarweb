# Zafarbek — лендинг с приёмом заявок

React + TypeScript + Vite + Tailwind CSS.

## Запуск

```bash
npm install
npm run dev
```

Откроется на http://localhost:5173

## Сборка

```bash
npm run build
npm run preview
```

## Структура проекта

```
src/
  App.tsx                 — роутинг (сайт / #admin) + провайдеры
  main.tsx                — точка входа
  index.css                — Tailwind + анимации терминала
  types.ts                  — общие типы (Service, Order)

  data/
    constants.ts            — имя, телефон владельца, пароль админки
    defaultServices.ts       — услуги по умолчанию (можно менять из админки)

  i18n/
    translations.ts          — тексты интерфейса на RU / EN / UZ
    LanguageContext.tsx       — контекст языка + переключение

  context/
    DataContext.tsx           — услуги и заявки: CRUD + localStorage

  components/
    Header.tsx, Hero.tsx, TerminalHero.tsx,
    Services.tsx, ServiceCard.tsx, ServiceModal.tsx,
    About.tsx, Footer.tsx, LanguageSwitcher.tsx,
    PublicSite.tsx            — публичная страница целиком

    admin/
      AdminApp.tsx             — вход + панель
      AdminGate.tsx             — форма пароля
      AdminPanel.tsx             — вкладки «Услуги» / «Заявки»
      ServicesEditor.tsx          — добавление/изменение/удаление услуг и цен
      OrdersTable.tsx              — таблица заявок со сменой статуса
```

## Как это работает

- **Клик по услуге** на сайте открывает модальное окно с формой заявки именно по этой услуге (имя, контакт, описание задачи). После отправки показывается номер заявки.
- **Смена языка** (RU / EN / UZ) — переключатель в шапке сайта и на админке, тексты интерфейса меняются мгновенно.
- **Админ-панель** открывается по ссылке `#admin` внизу страницы (или `/#admin` в адресной строке). Вход по паролю.
  - Пароль по умолчанию: **`zafarbek2026`** — задан в `src/data/constants.ts`, обязательно смените перед реальным использованием.
  - Вкладка **«Услуги и цены»** — можно добавлять, редактировать и удалять услуги, менять цены.
  - Вкладка **«Заявки»** — список всех заявок с возможностью менять статус (Новая / В работе / Готово).

## Важно: как сейчас хранятся данные

Это **фронтенд-демо без сервера**. Услуги, заявки и пароль-сессия хранятся в `localStorage` / `sessionStorage` браузера:

- Заявки, отправленные с телефона клиента, **не появятся** в админке, если вы открываете её с другого устройства/браузера.
- Пароль админки лежит в коде (виден в devtools) — это подходит для личного/демо-использования, но не для защиты от направленной атаки.

### Чтобы данные были общими для всех устройств, нужен один из вариантов:

1. **Backend + БД** — API-эндпоинт (Node/Express, FastAPI и т.д.) с базой (Postgres, Supabase, Firebase), который принимает заявки и отдаёт список услуг.
2. **Уведомления в Telegram/email** — заявка из формы сразу летит вам в бота или почту (Formspree, EmailJS, свой webhook).
3. **No-code хранилище** — Google Sheets / Airtable через готовый сервис.

Пароль администратора в этом случае тоже стоит проверять на сервере, а не в браузере.

Могу подключить любой из этих вариантов к этой же форме и админке — скажите, что удобнее (свой сервер, Telegram-бот, Google Sheets и т.д.).
